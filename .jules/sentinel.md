## YYYY-MM-DD - [Fix Missing Authorization on SR Attachments]
**Vulnerability:** Insecure Direct Object Reference (IDOR) / Missing Authorization on attachment upload (POST) and download (GET) endpoints. Any authenticated user could upload/download files for any SR by guessing the ID.
**Learning:** The attachment endpoints (`/api/attachments` and `/api/attachments/[id]`) correctly verified authentication (via `withAuthAndRateLimit`) but failed to enforce authorization checks against the related SR. The `ensureCanUpdateSR` and `ensureCanReadSR` policy functions existed but were not invoked.
**Prevention:** Always verify that a user has the specific permission (Read/Update) on the parent resource (SR) before allowing operations on its child resources (attachments). Use the existing policy functions (`@/lib/policies`) for these checks.
