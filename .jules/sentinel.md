## 2024-03-07 - Add Missing Authorization on Comments API
**Vulnerability:** IDOR (Insecure Direct Object Reference) in `GET` and `POST` for `/api/srs/[id]/comments`. The API endpoints retrieved and created comments based solely on the `id` from the URL, without verifying if the requesting user actually had permission to read the underlying SR.
**Learning:** Even if `withAuthAndRateLimit` verifies the user is authenticated, explicit resource-level authorization checks like `ensureCanReadSR` must be invoked in the route handler. Relying on `findUnique` for ID lookup is insufficient for authorization in multi-tenant or role-based systems.
**Prevention:** Always verify the requester's access to the parent resource (`ensureCanReadSR(session.user, sr)`) before granting access to its child resources (like comments, attachments, or activities). Ensure `clientId` and `requesterId` are fetched from the database to supply the policy function with correct context.
