# Task: CI/CD 배포 파이프라인 스테이징/운영 분리 방안 수립 및 구현

## 📋 진행 상태

- [x] **Think** — 요구사항 분석 및 기존 코드 영향 파악
- [/] **Plan** — 구체적 수정/추가 단계 정의 및 방안 제안
- [ ] **Implement** — 코드 작성 및 리팩토링
- [ ] **Test** — 테스트·빌드 실행으로 검증
- [ ] **Summarize** — 결과 요약

## 🔍 기존 CI/CD 설정 분석 결과

1. `.github/workflows/deploy.yml` 파일에서 `main` 및 `dev` 브랜치 push 이벤트에 대해 빌드 및 배포를 수행하고 있음.
2. 현재 배포 타겟 서버 VM(`SERVER_HOST`)과 도메인(`lkindo.kr`)이 단일 환경 기준으로 하드코딩되어 작동 중.
3. 이 상태에서 별도의 test 환경 없이 배포를 진행하면 충돌이 발생하거나 운영 도메인에 즉시 반영됨.

## 🎯 향후 수정 방향

- 사용자가 선택한 방안에 따라 `deploy.yml` 및 배포 스크립트를 커스터마이징하여 테스트 배포 선행 후 운영 배포 승인 프로세스를 구축함.
