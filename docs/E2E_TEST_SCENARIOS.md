# E2E 테스트 시나리오 문서

## 개요

이 문서는 SR Management System의 E2E 테스트 시나리오를 정리합니다.

## 테스트 파일 목록

### 1. 기본 페이지 접근 (`01-basic.spec.ts`)
- 로그인 페이지 접근
- 회원가입 페이지 접근
- 인증되지 않은 사용자의 대시보드 접근 제한

### 2. 인증 (`02-auth.spec.ts`)
- 로그인 플로우
- 로그아웃 플로우
- 세션 관리

### 3. SR 목록 (`03-sr-list.spec.ts`)
- SR 목록 페이지 로드
- SR 목록 표시
- SR 상세 페이지 이동

### 4. SR 생성 (`04-sr-create.spec.ts`)
- SR 생성 다이얼로그 열기
- SR 생성 플로우 (스킵됨 - API 인증 문제)
- SR 생성 유효성 검증

### 5. SR 상세 (`05-sr-detail.spec.ts`)
- SR 상세 정보 표시
- SR 메타데이터 확인

### 6. SR 수정 (`06-sr-update.spec.ts`)
- SR 수정 다이얼로그 열기
- SR 상세 페이지에서 수정

### 7. SR 필터링 및 검색 (`07-sr-filter-search.spec.ts`)
- SR 목록 페이지 로드
- 검색 기능
- 상태 필터
- 우선순위 필터

### 8. 사용자 관리 (`08-user-management.spec.ts`)
- 사용자 목록 페이지 접근
- 사용자 검색 기능
- 사용자 역할 관리 버튼

### 9. 고객사 관리 (`09-client-management.spec.ts`)
- 고객사 목록 페이지 접근
- 고객사 등록 버튼
- 고객사 검색 기능

### 10. SR 접수 플로우 (`10-sr-intake-flow.spec.ts`)
- SR 접수 화면 접근
- SR 접수 다이얼로그 열기
- SR 접수 정보 입력

### 11. SR 댓글 및 첨부파일 (`11-sr-comment-attachment.spec.ts`)
- SR 상세 페이지에서 댓글 섹션 확인
- 댓글 작성 기능
- 첨부파일 섹션 확인
- 첨부파일 업로드 버튼 확인

### 12. 역할 관리 (`12-role-management.spec.ts`)
- 역할 목록 페이지 접근
- 역할 등록 버튼
- 역할 상세 정보 확인
- 역할 권한 관리 버튼

### 13. SR 상태 변경 워크플로우 (`13-sr-status-workflow.spec.ts`)
- SR 상태 변경 버튼 확인
- SR 진행중 상태로 변경
- SR 완료 상태로 변경

### 14. 대시보드 개요 (`14-dashboard-overview.spec.ts`)
- 대시보드 페이지 접근
- SR 통계 카드 확인
- 최근 SR 목록 확인
- 빠른 액션 버튼 확인

### 15. 페이지네이션 및 정렬 (`15-pagination-sorting.spec.ts`)
- SR 목록 페이지네이션
- SR 목록 정렬 기능
- 사용자 목록 페이지네이션
- 고객사 목록 정렬

### 16. 사용자 프로필 관리 (`16-user-profile-management.spec.ts`)
- 프로필 페이지 접근
- 프로필 수정 버튼
- 비밀번호 변경 섹션
- 사용자 정보 표시

## 주요 사용자 시나리오

### 시나리오 1: SR 생성부터 완료까지
1. 사용자 로그인
2. SR 생성
3. MANAGER가 SR 접수
4. ENGINEER가 SR 처리
5. SR 완료 처리

### 시나리오 2: 사용자 관리
1. 관리자 로그인
2. 사용자 목록 조회
3. 새 사용자 등록
4. 사용자 역할 할당
5. 사용자 활성화/비활성화

### 시나리오 3: 고객사 관리
1. 관리자 로그인
2. 고객사 목록 조회
3. 새 고객사 등록
4. 고객사 담당자 배정
5. 고객사 서비스 카테고리 설정

### 시나리오 4: SR 필터링 및 검색
1. SR 목록 페이지 접근
2. 상태별 필터링
3. 우선순위별 필터링
4. 검색어로 SR 찾기
5. 정렬 기능 사용

### 시나리오 5: SR 댓글 및 첨부파일
1. SR 상세 페이지 접근
2. 댓글 작성
3. 첨부파일 업로드
4. 댓글 및 첨부파일 확인

## 테스트 실행 방법

```bash
# 모든 E2E 테스트 실행
npm run test:e2e

# UI 모드로 실행
npm run test:e2e:ui

# 디버그 모드로 실행
npm run test:e2e:debug

# 특정 테스트 파일 실행
npx playwright test e2e/10-sr-intake-flow.spec.ts
```
